    Introduction

    Welcome to the WeConnect API Reference! You can use our API to access WeConnect API endpoints, which can get information on various businesses and their reviews in our database.

    Businesses

    Create a Business

    HTTP Request:

    {
  "name": "somename",
  "address": "some address",
  "location": "some state",
  "category": "some business category"
}

    HTTP Response:

    {
  "id": 1,
  "name": "somename",
  "address": "some address",
  "location": "some state",
  "category": "some business category",
  "review": "some business review"
}

    This endpoint creates a business entity.

    HTTP Request

    POST http://example.com/v1/businesses

    HTTP Response

    STATUS - 201 Created
    BODY - application/json

    Get all Businesses

    HTTP Response:

    {
  "businessDataModels": [
      {
        "id": 1,
        "name": "SomeName",
        "address": "SomeAddress",
        "location": "SomeLocation",
        "category": "SomeCategory",
        "review": []
      },
      {
        "id": 2,
        "name": "SomeName",
        "address": "SomeAddress",
        "location": "SomeLocation",
        "category": "SomeCategory",
        "review": []
      }
    ],
  "error": false
}

    This endpoint retrieves all business entity.

    HTTP Request

    GET http://example.com/v1/businesses

    Queries

    By Location: GET http://example.com/v1/businesses?location=_location
    By Category: GET http://example.com/v1/businesses?category=_category

    HTTP Response

    STATUS - 200 Ok
    BODY - application/json

    Parameter Value Description
    _location location to query If supplied, the result contains only businesses from the location.
    _category category to query If supplied, the result contains only businesses from that category.

    Get a Specific Business

    HTTP Response:

    {
  "businessDataModel": {
    "id": <Id>,
    "name": "SomeName",
    "address": "SomeAddress",
    "location": "SomeLocation",
    "category": "SomeCategory",
    "review": []
  },
  "message": "success",
  "error": false
}

    This endpoint returns a specific business.

    HTTP Request

    GET http://example.com/v1/businesses/<Id>

    HTTP Response

    STATUS - 200 Ok
    BODY - application/json

    URL Parameters

    Parameter Description
    Id The Id of the business to return

    Update a Specific Business

    HTTP Request:

    {
  "name": "SomeName",
  "address": "SomeAddress",
  "location": "SomeLocation",
  "category": "SomeCategory"
}

    HTTP Response:

    {
  "businessDataModel": {
    "id": <Id>,
    "name": "SomeName",
    "address": "SomeAddress",
    "location": "SomeLocation",
    "category": "SomeCategory",
    "review": []
  },
  "message": "Business Updated",
  "error": false
}

    This endpoint updates a specific business.

    HTTP Request

    PUT http://example.com/v1/businesses/<Id>

    HTTP Response

    STATUS - 200 Ok
    BODY - application/json

    URL Parameters

    Parameter Description
    Id The Id of the business to be updated

    Reviews

    Review a Business

    HTTP Request:

    {
  "title": "some title",
  "description": "some description"
}

    HTTP Response:

    {
  "id": 1,
  "title": "some title",
  "description": "some description"
}

    This endpoint creates a review for a business.

    HTTP Request

    POST http://example.com/v1/businesses/<Id>/reviews

    HTTP Response

    STATUS - 201 Created
    BODY - application/json

    Parameter Description
    Id The Id of the business to be reviewed

    Get Reviews

    HTTP Response:

    {
  "reviews": [
      {
        "id": 1,
        "title": "some title",
        "description": "some description"
      },
      {
        "id": 2,
        "title": "some title",
        "description": "some description"
      }
    ],
  "message": "success",
  "error": false
}

    This endpoint retrieves all reviews for a business entity.

    HTTP Request

    GET http://example.com/v1/businesses/<Id>/reviews

    HTTP Response

    STATUS - 200 Ok
    BODY - application/json

    Parameter Description
    Id The Id of the business reviewed

  "password": "user password"
}

    HTTP Response:

    {
  "id": 1,
  "name": "user name",
  "email": "user email",
  "password": "user password"
}

    HTTP Request

  "password": "user password"

    HTTP Response:

  "error": false

    HTTP Request

    GET http://example.com/v1/auth/login

    HTTP Response

    STATUS - 200 Ok
    BODY - application/json

    Errors

    The WeConnect API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    404 Not Found -- The specified kitten could not be found.
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
